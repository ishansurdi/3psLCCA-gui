# Country-level Social Cost of Carbon - Database & Explorer

This repository contains the database and an interactive desktop explorer for the country-level Social Cost of Carbon (CSCC) estimates published in:

> Ricke, K., Drouet, L., Caldeira, K., & Tavoni, M. (2018). **Country-level social cost of carbon.** *Nature Climate Change*, 8, 895–900.

---

## Contents

| File | Description |
|---|---|
| `cscc_db_v2.csv` | Full database (247,860 rows, canonical source of truth) |
| `cscc_db.pkl` | Pandas pickle of the database with a 9-level MultiIndex for fast lookups (generated by `build_pickle.py`) |
| `build_pickle.py` | One-time script to convert the CSV to the pickle |
| `cscc_explorer.py` | Interactive GUI explorer (tkinter + pandas) |
| `test.py` | Verification script - checks that pickle values match the CSV |
| `archive/cscc_full_v1.csv` | Original v1 database published with the paper |

## Quick start

**Step 1 - build the pickle** (only needed once, or after the CSV changes):

```
python build_pickle.py
```

**Step 2 - launch the explorer:**

```
python cscc_explorer.py
```

**Run tests** to verify the pickle matches the CSV:

```
python test.py
```

## Database schema

Each row in `cscc_db_v2.csv` represents one unique combination of model, uncertainty, scenario, country, and discounting assumptions.

### Column reference

| Column | Values | Description |
|---|---|---|
| `run` | `bhm_sr`, `bhm_richpoor_sr`, `bhm_lr`, `bhm_richpoor_lr`, `djo_richpoor` | Climate damage model (see below) |
| `dmgfuncpar` | `bootstrap`, `estimates` | Whether damage function uncertainty is included |
| `climate` | `expected`, `uncertain` | Whether climate projection uncertainty is included |
| `SSP` | `SSP1`–`SSP5` | Shared Socioeconomic Pathway |
| `RCP` | `rcp45`, `rcp60`, `rcp85` | Representative Concentration Pathway |
| `N` | integer | Number of simulation runs used to compute the percentiles |
| `ISO3` | ISO 3166-1 alpha-3 | Country code; `WLD` = global aggregate |
| `prtp` | `1`, `2`, `NA` | Pure rate of time preference (%; `NA` when fixed discounting is used) |
| `eta` | `0p7`, `1p5`, `NA` | Elasticity of marginal utility (`NA` when fixed discounting is used) |
| `dr` | `3`, `5`, `NA` | Fixed discount rate (%; `NA` when Ramsey-rule discounting is used) |
| `16.7%` | float | Low CSCC estimate (16.7th percentile) in USD / tCO₂ |
| `50%` | float | Median CSCC estimate in USD / tCO₂ |
| `83.3%` | float | High CSCC estimate (83.3rd percentile) in USD / tCO₂ |

> The three statistics are the **median of the posterior distribution** of each percentile after Bayesian bootstrapping - e.g. the `16.7%` column is the median of the posterior distribution of the 16.7th percentile across simulations. The 16.7%–83.3% range forms a **66.7% confidence interval**.

---

### Damage models (`run`)

| Value | Reference | Specification |
|---|---|---|
| `bhm_sr` | Burke, Hsiang & Miguel (2015) | Short-run (no lag), pooled |
| `bhm_richpoor_sr` | Burke, Hsiang & Miguel (2015) | Short-run, income-dependent (rich/poor split) |
| `bhm_lr` | Burke, Hsiang & Miguel (2015) | Long-run (5-lag), pooled |
| `bhm_richpoor_lr` | Burke, Hsiang & Miguel (2015) | Long-run, income-dependent (rich/poor split) |
| `djo_richpoor` | Dell, Jones & Olken (2012) | Alternative impact function, income-dependent |

The rich/poor income threshold is **$2,268.53 (BHM)** and **$2,449.36 (DJO)** (2005 USD, updated in v2).

---

### Discounting

Two discounting frameworks are available, controlled jointly by `prtp`, `eta`, and `dr`:

**Growth-adjusted (Ramsey rule)** - `dr = NA`

| Label | `prtp` | `eta` |
|---|---|---|
| Growth-adjusted (prtp=2%, η=1.5) | `2` | `1p5` |
| Growth-adjusted (prtp=1%, η=1.5) | `1` | `1p5` |
| Growth-adjusted (prtp=2%, η=0.7) | `2` | `0p7` |
| Growth-adjusted (prtp=1%, η=0.7) | `1` | `0p7` |

**Fixed rate** - `prtp = NA`, `eta = NA`

| Label | `dr` |
|---|---|
| Fixed 3% | `3` |
| Fixed 5% | `5` |

### SSP / RCP default pairings

The paper uses the closest matching RCP for each SSP. These are the defaults in the explorer:

| SSP | Default RCP | Approximate warming (2100) |
|---|---|---|
| SSP1 | RCP6.0 | ~3°C |
| SSP2 | RCP6.0 | ~3°C |
| SSP3 | RCP8.5 | ~4.5°C |
| SSP4 | RCP6.0 | ~3°C |
| SSP5 | RCP8.5 | ~4.5°C |

The explorer also allows overriding the RCP independently of the SSP.

## Explorer - controls reference

| Control | What it does |
|---|---|
| **Country (ISO3)** | Select any of the 170 countries or `WLD` for the global aggregate |
| **SSP buttons** | Choose the socioeconomic pathway; defaults to SSP2 |
| **RCP override** | Force a specific RCP instead of the SSP-default pairing |
| **Damage function** | Switch between the four BHM specifications |
| **Discounting** | Choose discounting framework and parameters |
| **Damage params** | `bootstrap` includes full damage function uncertainty; `estimates` uses central parameters only |
| **Climate** | `expected` uses central climate projections; `uncertain` adds climate bootstrapping |
| **Percentile** | Choose which estimate to display prominently: 16.7% (optimistic), 50% (central), 83.3% (pessimistic) |

The full 66.7% confidence interval `[16.7% ; 83.3%]` is always shown below the main value regardless of which percentile is selected.

The status bar at the bottom shows the exact query parameters sent to the database for the current selection.

## Implementation notes

### Pickle vs CSV

`cscc_db.pkl` stores the DataFrame with a 9-column sorted MultiIndex (`ISO3`, `run`, `dmgfuncpar`, `climate`, `SSP`, `RCP`, `prtp`, `eta`, `dr`). This gives O(log n) lookups via `df.loc[key]` and loads ~3× faster than parsing the CSV.

**Important:** `build_pickle.py` passes `keep_default_na=False` to `pd.read_csv`. Without this, pandas converts the string `"NA"` in `prtp`, `eta`, and `dr` to float `NaN`, which silently breaks all Ramsey-rule lookups. The CSV is the canonical source - regenerate the pickle with `python build_pickle.py` if you ever edit the CSV.

### Test suite

`test.py` samples ~200 evenly-spaced rows from the raw CSV and looks each up in the pickle, comparing values with `math.isclose(rel_tol=1e-9)` to tolerate float64 string-parsing rounding. It exits with code 1 if any real mismatch is found.
